#TitleVideo Encrypeter and Decrypter - Pixel based Encrypting

A secure video processing pipeline was designed in which an input MP4 file is decoded into individual frames using FFmpeg. Each frame is subjected to a two-layer, key-driven transformation mechanism. The first layer applies a byte-wise XOR operation on pixel intensities using a cryptographic key derived via SHA-256 hashing, ensuring efficient value-level obfuscation. The second layer performs deterministic pixel permutation based on a pseudo-random sequence initialized through a key-derived seed, enabling reversible spatial scrambling. The transformed frames are subsequently re-encoded into an encrypted video stream. During playback, a custom rendering module performs real-time inverse operations—namely, seeded unshuffling followed by XOR reversal—using the same key, thereby reconstructing the original visual content dynamically in memory without persistent decryption, ensuring controlled and secure media access.
